Build a **PRO-LEVEL Python Web Panel (V3)** that runs inside a restricted environment like Pterodactyl (no root, no sudo). The system must act as a **Python process manager + web dashboard + Discord-controlled panel**, and must expose itself using a **Cloudflare Tunnel with token (user’s Cloudflare account)**.

---

# TECH STACK (STRICT)

* Backend: **FastAPI**
* Realtime: **WebSockets**
* Frontend: **Jinja2 (NO React)**
* Styling: **TailwindCSS (CDN)**
* Database: JSON (`database.json`, `config.json`)
* Process control: `subprocess`, `os`, `signal`
* Monitoring: `psutil`
* Discord Bot: **discord.py**

---

# CORE FEATURES

## 1. Script Manager

* Load `.py` files from `/scripts`

* Start / Stop / Restart scripts

* Use:
  subprocess.Popen(["python3", script])

* Store:

```json
{
  "script.py": {
    "pid": 1234,
    "status": "running",
    "start_time": 1710000000,
    "restarts": 0
  }
}
```

---

## 2. Smart Process Tracking

* Persist state in JSON
* On startup:

  * Check if PID exists
  * Fix broken states
* Show:

  * PID
  * Status
  * Uptime
  * Restart count

---

## 3. Live Logs (WebSocket)

* Endpoint:
  /ws/logs/{script}

* Stream logs from:
  /logs/<script>.log

* Features:

  * Auto-scroll
  * Reconnect
  * Real-time updates

---

## 4. Live System Stats

* Endpoint:
  /ws/stats

* Send every 2 seconds:

  * CPU %
  * RAM %
  * Active scripts

---

## 5. Web Panel UI

* Dashboard:

  * Script cards
  * 🟢 Running / 🔴 Stopped
  * Buttons:

    * Start
    * Stop
    * Restart
    * Logs

* Top bar:

  * CPU
  * RAM
  * Active processes

---

## 6. Upload & Delete

* Upload `.py` files safely
* Save to `/scripts`
* Delete script (stop if running)

---

## 7. Auto Restart System

* If script crashes → restart automatically
* Increase restart counter

---

## 8. REST API

* GET /api/scripts
* POST /api/start/{name}
* POST /api/stop/{name}
* POST /api/restart/{name}
* GET /api/status

# 🌐 CLOUDFLARE TUNNEL (TOKEN)

## Config File

config.json:

```json
{
  "cloudflare_token": "",
  "discord_token": ""
}
```

---

## Tunnel Requirements

* Auto-download `cloudflared` binary if missing

* Run:
  ./cloudflared tunnel run --token TOKEN

* Store:

  * PID
  * Status

* Auto restart if crashes

* Save logs:
  /logs/tunnel.log

---

## UI Display

* Tunnel status:
  🟢 Connected / 🔴 Stopped
* Show domain (user domain)

---

## Settings Page

* Input:

  * Cloudflare Token
  * Discord Bot Token
